# User Journeys

## Journey 1: Quick Voice Question
1. User opens app.
2. User taps talk button.
3. User asks a question naturally.
4. System transcribes input.
5. Assistant responds in natural spoken language.
6. Conversation is saved to history.

## Journey 2: Ask Assistant to Take Action
1. User asks assistant to do a task.
2. Assistant identifies required tool/action.
3. Assistant asks for approval if action is sensitive.
4. User approves.
5. Assistant executes action.
6. Assistant reports outcome conversationally.

## Journey 3: Continue Conversation Across Devices
1. User starts conversation on mobile.
2. Context is saved to backend.
3. User later opens Windows app.
4. Session history and memory are available.
5. User continues same assistant relationship seamlessly.
